## 1. 设定参数

```
head_dim = 4          # 每个注意力头的维度（实际中通常是 64/128）
seq_len = 3           # 序列长度，3 个 token
rope_base = 1000000   # 频率基数（和项目配置一致）
position_idx = None   # 不使用特殊的位置索引
```

假设原始输入经过 q_proj 后，某个头的 Q 和 K 为：

```
位置 0 的 Q: q₀ = [1.0,  2.0,  3.0,  4.0]
位置 1 的 Q: q₁ = [5.0,  6.0,  7.0,  8.0]
位置 2 的 Q: q₂ = [9.0, 10.0, 11.0, 12.0]

位置 0 的 K: k₀ = [0.1, 0.2, 0.3, 0.4]
位置 1 的 K: k₁ = [0.5, 0.6, 0.7, 0.8]
位置 2 的 K: k₂ = [0.9, 1.0, 1.1, 1.2]
```

Q 和 K 都是 4 维向量。RoPE 的核心是：**对每一对维度 (x₂ᵢ, x₂ᵢ₊₁) 施加一个二维旋转**。

---

## 2. 步骤 1：生成基础频率 (θ)

### 代码对应

```python
# torch.arange(0, dim, 2) = [0, 2]
# [: (dim // 2)] = [0, 2]（取前 dim//2 = 2 个）

freqs = 1 / (rope_base ** (torch.arange(0, dim, 2).float() / dim))
```

### 手工计算

```
dim = 4, 所以 dim // 2 = 2

arange(0, 4, 2) = [0, 2]
除以 dim: [0/4, 2/4] = [0.0, 0.5]

θ₀ = 1 / (1,000,000 ^ 0.0) = 1 / 1 = 1.0
θ₁ = 1 / (1,000,000 ^ 0.5) = 1 / 1000 = 0.001

所以 freqs = [1.0, 0.001]
```

**结果**：

| 维度对 | 频率 θ |
|--------|--------|
| 第 0 对 (dim 0, 1) | θ₀ = **1.0** |
| 第 1 对 (dim 2, 3) | θ₁ = **0.001** |

> **关键观察**：θ₀ ≫ θ₁。第 0 对维度旋转很快（高频率），第 1 对维度旋转很慢（低频率）。这对应了 "高频捕捉局部关系，低频捕捉远距离关系" 的设计理念。

## 3. 步骤 2：计算每个位置的旋转角度

### 代码对应

```python
t = torch.arange(end)           # [0, 1, 2]
freqs = torch.outer(t, freqs)   # 外积
```

### 外积计算

```
        θ₀=1.0   θ₁=0.001
      ┌────────┬────────┐
pos=0 │ 0×1.0  │ 0×0.001│
pos=1 │ 1×1.0  │ 1×0.001│
pos=2 │ 2×1.0  │ 2×0.001│
      └────────┴────────┘
```

**结果矩阵**（每个位置、每个维度对的旋转角度）：

| 位置 | 第 0 对角度 | 第 1 对角度 |
|------|------------|------------|
| pos=0 | 0.0 | 0.0 |
| pos=1 | 1.0 | 0.001 |
| pos=2 | 2.0 | 0.002 |

## 4. 步骤 3：生成 cos 和 sin

### 代码对应

```python
freqs_cos = torch.cat([torch.cos(freqs), torch.cos(freqs)], dim=-1)
freqs_sin = torch.cat([torch.sin(freqs), torch.sin(freqs)], dim=-1)
```

### 计算 cos 和 sin

**对于 freqs 矩阵**（3 行 2 列）：

```
freqs = [[0.0,    0.0   ],
         [1.0,    0.001 ],
         [2.0,    0.002 ]]
```

**计算 cos**（注意：这里 freqs 只有 2 列，cat 两份后变成 4 列）：

```
pos=0: cos(0.0)=1.00000,  cos(0.0)=1.00000  → [1.00000, 1.00000, 1.00000, 1.00000]
pos=1: cos(1.0)=0.54030,  cos(0.001)≈0.9999995 → [0.54030, 0.9999995, 0.54030, 0.9999995]
pos=2: cos(2.0)=-0.41615, cos(0.002)≈0.999998  → [-0.41615, 0.999998, -0.41615, 0.999998]
```

**计算 sin**：

```
pos=0: sin(0.0)=0.00000,  sin(0.0)=0.00000  → [0.00000, 0.00000, 0.00000, 0.00000]
pos=1: sin(1.0)=0.84147,  sin(0.001)≈0.001  → [0.84147, 0.001, 0.84147, 0.001]
pos=2: sin(2.0)=0.90930,  sin(0.002)≈0.002  → [0.90930, 0.002, 0.90930, 0.002]
```

**最终结果**（注意这里已经 cat 过了，每行 4 列）：

```
cos = [[1.00000,  1.00000,  1.00000,  1.00000 ],    # pos=0
       [0.54030,  1.00000,  0.54030,  1.00000 ],    # pos=1
       [-0.41615, 1.00000, -0.41615, 1.00000 ]]    # pos=2

sin = [[0.00000,  0.00000,  0.00000,  0.00000 ],    # pos=0
       [0.84147,  0.001,    0.84147,  0.001   ],    # pos=1
       [0.90930,  0.002,    0.90930,  0.002   ]]    # pos=2
```

> **为什么 cat 两份？**
>
> 配对方式是 **(xᵢ, xᵢ₊d/₂)**，不是相邻配对。向量被切成两半，前半截第 i 个和后半截第 i 个组成一对。
>
> `rotate_half` 把后半截取负后拼到前面，为了让逐元素乘法时，位置 i 和位置 i+d/2 都能乘到同一个 cos(θᵢ)/sin(θᵢ)，cos 和 sin 需要复制一份。
>
> 详细展开见下节。

## 5. 步骤 4：应用到 Q 和 K

### 5.1 rotate_half 的作用

```python
def rotate_half(x):
    return torch.cat(
        (-x[..., x.shape[-1] // 2 :], x[..., : x.shape[-1] // 2]),
        dim=-1
    )
```

对 `[a, b, c, d]` 应用 `rotate_half`（d=4，配对为 (a,c) 和 (b,d)）：

```
输入:  [a,     b,     c,     d    ]
        └─前半截─┘  └─后半截─┘
        │第0对起始│  │第0对另一半│
        │第1对起始│  │第1对另一半│

后半截取负: [-c,    -d]
前半截不变:  [a,     b]
拼接结果:   [-c,    -d,    a,     b    ]
            └─后半截取负放到前面─┘└─原前半截接到后面─┘
```

### 5.2 具体计算

RoPE 公式：

```
q' = q * cos + rotate_half(q) * sin
```

逐元素展开（d=4，配对为 (q₀,q₂) 用 θ₀，(q₁,q₃) 用 θ₁）：

```
q'[0] = q[0] * cos[0] + (-q[2]) * sin[0]   ← 第 0 对 (q₀,q₂) 的前半：q₀*cos(θ₀) - q₂*sin(θ₀)
q'[1] = q[1] * cos[1] + (-q[3]) * sin[1]   ← 第 1 对 (q₁,q₃) 的前半：q₁*cos(θ₁) - q₃*sin(θ₁)
q'[2] = q[2] * cos[2] + q[0] * sin[2]     ← 第 0 对 (q₀,q₂) 的后半：q₂*cos(θ₀) + q₀*sin(θ₀)
q'[3] = q[3] * cos[3] + q[1] * sin[3]     ← 第 1 对 (q₁,q₃) 的后半：q₃*cos(θ₁) + q₁*sin(θ₁)
```

注意 `cos[2]=cos[0]=cos(θ₀)`，`cos[3]=cos[1]=cos(θ₁)`，因为 cos 是复制了两份的 `[cos(θ₀), cos(θ₁), cos(θ₀), cos(θ₁)]`。

整理后：

```
q'[0] = q[0] * cos(θ₀) - q[2] * sin(θ₀)
q'[1] = q[1] * cos(θ₁) - q[3] * sin(θ₁)
q'[2] = q[2] * cos(θ₀) + q[0] * sin(θ₀)
q'[3] = q[3] * cos(θ₁) + q[1] * sin(θ₁)
```

这正是二维旋转矩阵的标准形式！

### 5.3 位置 0 的 Q（基准，不旋转）

```
q₀ = [1.0, 2.0, 3.0, 4.0]
cos[0] = [1.0, 1.0, 1.0, 1.0]
sin[0] = [0.0, 0.0, 0.0, 0.0]

rotate_half(q₀) = [-3.0, -4.0, 1.0, 2.0]

q₀' = q₀ * cos[0] + rotate_half(q₀) * sin[0]
    = [1.0*1.0, 2.0*1.0, 3.0*1.0, 4.0*1.0] + [-3.0*0.0, -4.0*0.0, 1.0*0.0, 2.0*0.0]
    = [1.0, 2.0, 3.0, 4.0]
```

> 位置 0 的旋转角度都是 0，所以 **q₀' = q₀**，保持不变。

### 5.4 位置 1 的 Q

```
q₁ = [5.0, 6.0, 7.0, 8.0]
cos[1] = [0.54030, 1.00000, 0.54030, 1.00000]
sin[1] = [0.84147, 0.001,   0.84147, 0.001  ]

rotate_half(q₁) = [-7.0, -8.0, 5.0, 6.0]

q₁' = q₁ * cos[1] + rotate_half(q₁) * sin[1]

计算每一项：
q₁'[0] = 5.0 * 0.54030 + (-7.0) * 0.84147
       = 2.7015 - 5.8903
       = -3.1888

q₁'[1] = 6.0 * 1.00000 + (-8.0) * 0.001
       = 6.0 - 0.008
       = 5.992

q₁'[2] = 7.0 * 0.54030 + 5.0 * 0.84147
       = 3.7821 + 4.2074
       = 7.9895

q₁'[3] = 8.0 * 1.00000 + 6.0 * 0.001
       = 8.0 + 0.006
       = 8.006

q₁' ≈ [-3.1888, 5.992, 7.9895, 8.006]
```

### 5.5 位置 2 的 Q

```
q₂ = [9.0, 10.0, 11.0, 12.0]
cos[2] = [-0.41615, 1.00000, -0.41615, 1.00000]
sin[2] = [0.90930,  0.002,   0.90930,  0.002  ]

rotate_half(q₂) = [-11.0, -12.0, 9.0, 10.0]

q₂'[0] = 9.0 * (-0.41615) + (-11.0) * 0.90930
       = -3.7454 - 10.0023
       = -13.7477

q₂'[1] = 10.0 * 1.00000 + (-12.0) * 0.002
       = 10.0 - 0.024
       = 9.976

q₂'[2] = 11.0 * (-0.41615) + 9.0 * 0.90930
       = -4.5777 + 8.1837
       = 3.606

q₂'[3] = 12.0 * 1.00000 + 10.0 * 0.002
       = 12.0 + 0.02
       = 12.02

q₂' ≈ [-13.7477, 9.976, 3.606, 12.02]
```

### 5.6 最终结果汇总

```
原始 Q:
  q₀ = [ 1.0,    2.0,   3.0,    4.0   ]
  q₁ = [ 5.0,    6.0,   7.0,    8.0   ]
  q₂ = [ 9.0,   10.0,  11.0,   12.0   ]

旋转后 Q':
  q₀' = [ 1.0,    2.0,   3.0,    4.0   ]     ← 位置 0，角度 0，不变
  q₁' = [-3.1888, 5.992, 7.9895, 8.006  ]     ← 位置 1，旋转了
  q₂' = [-13.7477, 9.976, 3.606, 12.02 ]     ← 位置 2，旋转更多
```

**K 的计算方式完全相同**，只是输入换成 K 的值。

## 6. 步骤 5：验证相对位置性质

RoPE 的核心特性：**旋转后的点积只取决于相对位置差**。

### 6.1 验证 q₀' · k₀'

位置 0 和位置 0（相对差 = 0）：

```
q₀' = [1.0, 2.0, 3.0, 4.0]
k₀' = [0.1, 0.2, 0.3, 0.4]

q₀' · k₀' = 1.0*0.1 + 2.0*0.2 + 3.0*0.3 + 4.0*0.4
          = 0.1 + 0.4 + 0.9 + 1.6
          = 3.0
```

### 6.2 验证 q₁' · k₁'

位置 1 和位置 1（相对差 = 0）：

```
q₁' ≈ [-3.1888, 5.992, 7.9895, 8.006]
k₁' 的值类似计算...

k₁'[0] = 0.5*0.54030 + (-0.7)*0.84147 = 0.2702 - 0.5890 = -0.3188
k₁'[1] = 0.6*1.0 + (-0.8)*0.001 = 0.6 - 0.0008 = 0.5992
k₁'[2] = 0.7*0.54030 + 0.5*0.84147 = 0.3782 + 0.4207 = 0.7989
k₁'[3] = 0.8*1.0 + 0.6*0.001 = 0.8 + 0.0006 = 0.8006

q₁' · k₁' = (-3.1888)*(-0.3188) + 5.992*0.5992 + 7.9895*0.7989 + 8.006*0.8006
          ≈ 1.016 + 3.590 + 6.383 + 6.410
          ≈ 17.40
```

### 6.3 验证 q₁' · k₀'

位置 1 和位置 0（相对差 = 1）：

```
q₁' ≈ [-3.1888, 5.992, 7.9895, 8.006]
k₀' = [0.1, 0.2, 0.3, 0.4]

q₁' · k₀' = (-3.1888)*0.1 + 5.992*0.2 + 7.9895*0.3 + 8.006*0.4
          ≈ -0.319 + 1.198 + 2.397 + 3.202
          ≈ 6.48
```

### 6.4 验证 q₂' · k₁'

位置 2 和位置 1（相对差 = 1）：

```
q₂' ≈ [-13.7477, 9.976, 3.606, 12.02]
k₁' ≈ [-0.3188, 0.5992, 0.7989, 0.8006]

q₂' · k₁' = (-13.7477)*(-0.3188) + 9.976*0.5992 + 3.606*0.7989 + 12.02*0.8006
          ≈ 4.383 + 5.978 + 2.881 + 9.623
          ≈ 22.87
```

### 6.5 为什么这验证了相对位置性质？

这里用具体的数值不容易直接看出相对性质，因为原始 Q/K 的值本身不同。

**真正的验证方式**：如果我们让 `q₁ = q₀` 且 `k₁ = k₀`，那么 `q₁' · k₁'` 应该等于 `q₀' · k₀'`。

更严谨地说，RoPE 保证的是：

```
如果 q_pos 和 k_pos 的"形状"相同（即经过 RoPE 后），
那么 q_m' · k_n' 只取决于 (m - n)，不取决于 m 或 n 的绝对值。
```

这就是 **相对位置编码** 的核心优势 —— 模型学习的是"A 在 B 前面 3 个位置"，而不是"A 在第 5 个位置"。

## 8. 对比：为什么不用绝对位置编码

### 绝对位置编码（如正弦位置编码）

```python
# 原始 Transformer 论文的做法
PE(pos, 2i)   = sin(pos / 10000^(2i/d))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d))

# 然后直接把 PE 加到输入上
x = x + PE
```

**问题**：
- 模型学到的是"第 5 个位置的词"
- 如果训练时最长序列是 2048，测试时来了 3000 个 token，模型没见过这么长的位置编码，效果很差（**外推性差**）

### RoPE（相对位置编码）

```python
# RoPE 的做法：旋转 Q 和 K
q' = RoPE(q, pos)
k' = RoPE(k, pos)

# 点积结果只和相对位置有关
q_m' · k_n' = f(|m - n|)
```

**优势**：
- 模型学到的是"A 在 B 前面 3 个位置"
- 不管绝对位置是多少，只要相对距离相同，点积就相同
- 可以更好地外推到更长的序列（配合 YaRN 等扩展技术）

### 直观对比

| | 绝对位置编码 | RoPE |
|---|---|---|
| 编码方式 | 向量加法 | 向量旋转 |
| 学到的信息 | "我在第 5 位" | "我距离你 3 个位置" |
| 外推能力 | 差 | 好 |
| 和注意力的兼容性 | 需特殊处理 | 天然兼容 |
| 当前主流模型 | 基本不用 | LLaMA、Qwen、DeepSeek 等都用 |

## 总结

RoPE 的完整流程可以概括为一句话：

> **"对每个位置，用不同频率旋转 Q 和 K 向量的每一对维度，使得点积结果只与相对位置有关。"**

四个步骤：

1. **生成频率** `θ_i = base^(-2i/d)` — 不同维度对有不同的旋转速度
2. **计算角度** `angle = pos × θ` — 每个位置有独特的旋转角度
3. **生成 cos/sin** — 预计算避免重复三角函数
4. **应用旋转** `q' = q * cos + rotate_half(q) * sin` — 高效向量化实现

本项目中的 RoPE 还额外支持 **YaRN 扩展**（用于超长序列推理），在 `precompute_freqs_cis` 函数中通过 `rope_scaling` 参数控制，详见 [Attention_详解.md](Attention_详解.md) 第 6.6 节。
